# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest | ✅ |

## Reporting a Vulnerability

If you discover a security vulnerability, please do not open a public GitHub issue.

Instead, report it by emailing:

security@example.com

Include:
- A description of the vulnerability
- Steps to reproduce it
- Screenshots or proof of concept (if applicable)

We aim to acknowledge reports within 3 business days and provide updates during the investigation.

If the vulnerability is confirmed, we will release a security fix as soon as possible and credit the reporter (if they wish).
